1. Project Title & Introduction Overview

AccessMaps is a navigation tool focused on sidewalk-level accessibility. While traditional mapping applications optimize for speed or distance, they often ignore real-world barriers such as missing curb ramps, steep slopes, broken sidewalks, or temporary construction.

This project explores how a crowdsourced, accessibility-first mapping platform can help people with mobility limitations plan safer, more comfortable routes. Our goal is to demonstrate an end-to-end system‚Äîfrom mobile client to cloud backend‚Äîthat collects, stores, and surfaces accessibility information in a usable and scalable way.

2. Background & Motivation Why This Matters

For millions of people worldwide‚Äîwheelchair users, seniors, people with injuries, parents with strollers‚Äîroute choice is about more than distance. A single missing curb cut or steep hill can make an otherwise ‚Äúfast‚Äù route impossible.

However, detailed accessibility data is typically fragmented, outdated, or completely missing from standard navigation tools. City GIS teams may track some infrastructure, but conditions on the ground change faster than official datasets can be updated.

AccessMaps was motivated by:

  • A lack of accessibility-aware routing in mainstream map apps.
  • Growing interest in inclusive, human-centered urban design.
  • The potential of crowdsourcing to keep data fresh and localized.
Core Question: Can a community-driven mapping platform provide more relevant, sidewalk-level accessibility information than traditional navigation tools?

3. Methodology / Approach How We Built It

AccessMaps is implemented as a full-stack system: a SwiftUI-based iOS application backed by a FastAPI service hosted on AWS. The architecture is designed to be modular so that new data sources (e.g., city GIS data, machine learning models) can be integrated over time.

Process

  • User & Problem Exploration: Identified navigation pain points through informal interviews and review of accessibility advocacy resources.
  • System Design: Defined data models for accessibility reports, locations, and user profiles. Designed API routes and authentication flow.
  • Implementation: Built the iOS front-end in SwiftUI and the backend with FastAPI, deploying services to AWS.
  • Testing & Iteration: Validated endpoints, refined UI flows, and tested data persistence and retrieval.

Technologies & Tools

SwiftUI (iOS) FastAPI (Python) AWS Cognito AWS RDS (PostgreSQL) AWS S3 Docker Git & GitHub MapKit / Maps SDK

The iOS app handles user interactions and map rendering, while the backend manages authentication, data storage, and business logic. AWS provides scalable infrastructure for future expansion.

4. Results & Findings What We Built

The current AccessMaps prototype demonstrates the feasibility of a crowdsourced accessibility mapping platform. While the dataset is small at this stage, the main outcome is a working end-to-end system that enables users to:

  • Create an account and manage a basic profile.
  • View an interactive map focused on pedestrian navigation.
  • Submit accessibility reports (e.g., missing curb ramp, steep slope, surface issues).
  • Store and retrieve reports through a cloud-hosted backend.

5. Discussion & Future Work Reflection

Building AccessMaps reinforced how complex ‚Äúsimple‚Äù navigation really is when accessibility is included. Sidewalk conditions, curb ramps, and gradients vary block by block, and no single data source is sufficient.

The project also highlighted the importance of designing for gradual growth: the platform must still be useful with limited data, while making it easy for users to contribute more over time.

Limitations

  • Data coverage is limited to test areas and early pilot usage.
  • Route recommendations are not yet fully optimized using accessibility constraints.
  • No automated computer vision integration is live at this stage.

Future Work

  • Integrate city GIS datasets for sidewalks, crosswalks, and curb ramps.
  • Deploy computer vision models to detect ramps, slopes, and surface defects from images.
  • Implement customizable routing (e.g., ‚Äúavoid steep hills‚Äù, ‚Äúprefer curb ramps‚Äù).
  • Add gamification elements to encourage more accessibility reports.
  • Release an Android version and a lightweight web client.
